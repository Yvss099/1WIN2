Rappel du Jour 1

Un serveur a toujours une IP statique. DNS et DHCP n'ont de sens que sur une machine dont l'adresse ne bouge jamais : c'est elle que tout le réseau va interroger.

Une correction utile sur les retours à la ligne

Dans certains supports, les commandes longues sont coupées avec un antislash \ en fin de ligne. C'est la syntaxe Linux. En PowerShell, le caractère de continuation est l'accent grave ` (backtick). Toutes les commandes ci-dessous utilisent le bon caractère.

DNS — traduire les noms en adresses

La métaphore de l'annuaire

Le DNS (Domain Name System, « système de noms de domaine »), c'est l'annuaire téléphonique du réseau. Vous connaissez le nom de quelqu'un (google.com), mais pour l'appeler il vous faut son numéro (l'adresse IP). Le DNS est le standardiste qui, à chaque nom, répond par le bon numéro. Sans lui, il faudrait retenir des suites de chiffres pour joindre chaque machine.

On dit que le DNS fait de la résolution de noms : transformer srv-dc01.formation.local en 192.168.100.10. Il gère aussi le chemin inverse (numéro vers nom), garde en mémoire les réponses récentes (le cache, comme un post-it de numéros fréquents), et sait à qui demander quand il ne connaît pas (les redirecteurs).

Les types d'enregistrements

Chaque ligne de l'annuaire a un type selon ce qu'elle décrit :

TypeRôleDans la métaphore
ANom → adresse IPv4La ligne « nom → numéro » de base
AAAANom → adresse IPv6Le même, version IPv6
CNAMEAlias : un nom qui renvoie vers un autreUn surnom qui pointe vers le vrai nom
MXServeur de messagerie du domaine« Pour le courrier, adressez-vous à… »
PTRAdresse IP → nom (résolution inverse)L'annuaire inversé : on cherche par numéro
SRVLocalise un service (ex : un contrôleur de domaine)« Le service X se trouve à cette adresse »
À retenir

Les enregistrements SRV seront cruciaux au Jour 3 : c'est par eux que les clients trouvent automatiquement le contrôleur Active Directory. Retenez qu'ils existent.

Installer le rôle DNS

En interface graphique (Server Manager)

  1. Ouvrir Gestionnaire de serveur (Server Manager).
  2. Menu Gérer → Ajouter des rôles et fonctionnalités.
  3. Type d'installation : Installation basée sur un rôle ou une fonctionnalité.
  4. Sélectionner le serveur, puis cocher Serveur DNS dans la liste des rôles.
  5. Accepter l'ajout des outils d'administration, puis Installer.

En PowerShell (méthode recommandée)

Administrator: Windows PowerShell
# Installer le rôle DNS avec ses outils d'administration
Install-WindowsFeature -Name DNS -IncludeManagementTools

# Vérifier que le rôle est bien installé
Get-WindowsFeature -Name DNS

# Vérifier que le service tourne
Get-Service DNS

Analysons ces commandes ligne par ligne :

  • Install-WindowsFeature -Name DNS installe le rôle ; -IncludeManagementTools ajoute la console DNS Manager (dnsmgmt.msc).
  • Get-WindowsFeature confirme l'état Installed.
  • Get-Service DNS doit afficher Running.
Bonne pratique

En entreprise on installe toujours en PowerShell : c'est reproductible, scriptable, et identique sur 1 ou 50 serveurs. La GUI sert surtout à vérifier et à comprendre la structure.

Créer les zones DNS

La métaphore poursuivie

Une zone, c'est le carnet d'adresses d'un quartier : la portion de l'annuaire dont ce serveur est responsable. La zone directe classe par nom (nom → numéro). La zone inversée, c'est l'annuaire à l'envers : on connaît le numéro et on cherche le nom (numéro → nom).

Administrator: Windows PowerShell
# Zone DIRECTE : le carnet du domaine formation.local (nom -> IP)
Add-DnsServerPrimaryZone -Name "formation.local" -ZoneFile "formation.local.dns"

# Zone INVERSÉE : l'annuaire à l'envers pour le réseau 192.168.100.0/24 (IP -> nom)
Add-DnsServerPrimaryZone -NetworkID "192.168.100.0/24" -ZoneFile "100.168.192.in-addr.arpa.dns"

# Lister les zones créées
Get-DnsServerZone
Le piège de la zone inversée

Le nom de la zone inversée s'écrit avec les octets du réseau à l'envers, suivis de .in-addr.arpa. Pour 192.168.100.0/24, cela donne 100.168.192.in-addr.arpa. C'est contre-intuitif, mais c'est ainsi que le DNS range les adresses pour les chercher par numéro.

Ajouter les enregistrements

On remplit maintenant le carnet. Chaque commande ajoute une ligne d'un type précis.

Administrator: Windows PowerShell
# A : associe un nom à une IPv4
Add-DnsServerResourceRecordA -ZoneName "formation.local" `
                             -Name "srv-dc01" -IPv4Address "192.168.100.10"

# CNAME : "www" est un alias qui renvoie vers srv-web.formation.local
Add-DnsServerResourceRecordCName -ZoneName "formation.local" `
                                 -Name "www" -HostNameAlias "srv-web.formation.local"

# MX : pour le courrier de formation.local, s'adresser à mail.formation.local
Add-DnsServerResourceRecordMX -ZoneName "formation.local" `
                              -Name "." -MailExchange "mail.formation.local" -Preference 10

# PTR : dans la zone inversée, le numéro .10 correspond à srv-dc01
Add-DnsServerResourceRecordPtr -ZoneName "100.168.192.in-addr.arpa" `
                               -Name "10" -PtrDomainName "srv-dc01.formation.local"
CNAME : ne pas pointer vers une IP

Un CNAME renvoie vers un nom, jamais vers une adresse IP. -HostNameAlias attend donc srv-web.formation.local, pas 192.168.100.30. Le surnom pointe vers le vrai nom ; c'est le vrai nom (un enregistrement A) qui, lui, porte l'IP.

À propos du MX et de la Preference

-Preference 10 est une priorité : plus le nombre est bas, plus le serveur est prioritaire. Avec un seul serveur de courrier, la valeur importe peu ; elle sert quand il y en a plusieurs.

Redirecteurs et cache

Le standardiste qui appelle les renseignements

Votre DNS connaît formation.local, mais pas google.com. Plutôt que d'abandonner, il transmet la question à un redirecteur : un gros annuaire public (8.8.8.8 chez Google, 1.1.1.1 chez Cloudflare). C'est le réflexe « je ne l'ai pas, je demande aux renseignements ».

Administrator: Windows PowerShell
# Rediriger les noms inconnus vers des DNS publics
Set-DnsServerForwarder -IPAddress "8.8.8.8","1.1.1.1"
Get-DnsServerForwarder

# Régler le cache : durée max des réponses gardées en mémoire
Set-DnsServerCache -MaxTTL "1.00:00:00" -MaxNegativeTTL "00:15:00"
Le cache, c'est le post-it

MaxTTL 1.00:00:00 = garder une réponse jusqu'à 1 jour. MaxNegativeTTL 00:15:00 = se souvenir d'un échec 15 minutes seulement (au cas où le nom apparaîtrait bientôt). Le cache évite de redemander sans cesse la même chose : c'est l'optimisation principale du DNS.

TP guidé 1 — DNS pour formation.local

Objectif : monter un serveur DNS complet pour l'entreprise de formation. Durée : 60 min. Livrable : captures de la console DNS + résultats des tests.

Étape 1 — Installer et démarrer

Administrator: Windows PowerShell
Install-WindowsFeature -Name DNS -IncludeManagementTools
Get-Service DNS          # doit afficher Running

Étape 2 — Zones et enregistrements

Créez la zone directe, la zone inversée, puis les enregistrements demandés :

Administrator: Windows PowerShell
Add-DnsServerPrimaryZone -Name "formation.local" -ZoneFile "formation.local.dns"
Add-DnsServerPrimaryZone -NetworkID "192.168.100.0/24" -ZoneFile "100.168.192.in-addr.arpa.dns"

# Enregistrements A
Add-DnsServerResourceRecordA -ZoneName "formation.local" -Name "srv-dc01" -IPv4Address "192.168.100.10"
Add-DnsServerResourceRecordA -ZoneName "formation.local" -Name "srv-file" -IPv4Address "192.168.100.20"
Add-DnsServerResourceRecordA -ZoneName "formation.local" -Name "srv-web"  -IPv4Address "192.168.100.30"
Add-DnsServerResourceRecordA -ZoneName "formation.local" -Name "mail"     -IPv4Address "192.168.100.25"

# www = alias de srv-web ; MX pour le courrier
Add-DnsServerResourceRecordCName -ZoneName "formation.local" -Name "www" -HostNameAlias "srv-web.formation.local"
Add-DnsServerResourceRecordMX -ZoneName "formation.local" -Name "." -MailExchange "mail.formation.local" -Preference 10

Étape 3 — Redirecteurs

Administrator: Windows PowerShell
Set-DnsServerForwarder -IPAddress "8.8.8.8","1.1.1.1"
Resolve-DnsName google.com   # doit résoudre grâce aux redirecteurs

Étape 4 — Tests de validation

Administrator: Windows PowerShell
Resolve-DnsName srv-dc01.formation.local            # nom -> IP
Resolve-DnsName "192.168.100.10" -Type PTR          # IP -> nom (inverse)
Resolve-DnsName www.formation.local                 # suit l'alias vers srv-web
Resolve-DnsName formation.local -Type MX            # serveur de courrier
Test-DnsServer -IPAddress "127.0.0.1" -ZoneName "formation.local"
Critères de réussite

Les quatre Resolve-DnsName renvoient une réponse cohérente, la résolution inverse retrouve bien srv-dc01, et google.com se résout via les redirecteurs.

DHCP — distribuer les adresses automatiquement

La métaphore de l'hôtel

Le DHCP (Dynamic Host Configuration Protocol, « protocole de configuration dynamique des hôtes »), c'est la réception d'un hôtel. À son arrivée, un client n'a pas de chambre. Il demande à la réception, qui lui en attribue une de libre, pour une durée donnée (le bail), avec le petit livret d'accueil (les options : où est la sortie/passerelle, le numéro du standard/DNS…). Quand le client repart, la chambre redevient disponible.

Le dialogue DORA

L'attribution suit quatre échanges, dont on retient l'acronyme DORA :

D Discover Client -> "Quelqu'un peut me donner une adresse ?" (diffusion) O Offer Serveur -> "Je te propose 192.168.100.50" (proposition) R Request Client -> "D'accord, je prends 192.168.100.50" (demande) A Acknowledge Serveur -> "C'est noté" + options (DNS, passerelle) (confirmation)
Pourquoi un serveur ne prend jamais d'adresse DHCP

DHCP est fait pour les clients de passage (postes, portables). Un serveur, lui, est une chambre permanente : il garde son IP statique. On distribue par DHCP aux clients, jamais aux serveurs.

Installer le rôle DHCP

Administrator: Windows PowerShell
# Installer le rôle DHCP avec ses outils
Install-WindowsFeature DHCP -IncludeManagementTools
Get-Service DHCPServer        # doit être Running

# Redémarrer le service après configuration
Restart-Service DHCPServer
« Autorisation » DHCP : pour plus tard

Dans un domaine Active Directory, un serveur DHCP doit être autorisé avant de distribuer (sécurité anti-serveur pirate). Comme AD n'existe qu'au Jour 3, on travaille pour l'instant en mode autonome (hors domaine), où cette autorisation n'est pas requise. La commande Add-DhcpServerInDC sera vue au Jour 3.

Étendues, options et réservations

L'étage, le livret, la chambre attitrée

Une étendue (scope) est l'étage de chambres que la réception a le droit de distribuer (une plage d'adresses). Les options sont le livret d'accueil remis avec la chambre (passerelle, DNS, suffixe). Une réservation dit « la chambre 25 est toujours pour M. Imprimante » : la même IP pour la même machine, identifiée par son adresse MAC.

Créer une étendue et ses options

Administrator: Windows PowerShell
# L'étage de chambres distribuables : .50 à .200
Add-DhcpServerv4Scope -Name "Réseau Formation" `
                      -StartRange "192.168.100.50" `
                      -EndRange "192.168.100.200" `
                      -SubnetMask "255.255.255.0" `
                      -Description "Postes de travail"

# Le livret d'accueil (options), désigné par un numéro normalisé :
Set-DhcpServerv4OptionValue -ScopeId "192.168.100.0" -OptionId 3  -Value "192.168.100.1"          # 3  = passerelle
Set-DhcpServerv4OptionValue -ScopeId "192.168.100.0" -OptionId 6  -Value "192.168.100.10","8.8.8.8" # 6  = serveurs DNS
Set-DhcpServerv4OptionValue -ScopeId "192.168.100.0" -OptionId 15 -Value "formation.local"          # 15 = suffixe DNS

# Ouvrir l'étage (activer l'étendue)
Set-DhcpServerv4Scope -ScopeId "192.168.100.0" -State Active

Analysons les options clés :

  • Option 3 — la passerelle par défaut : la sortie de l'immeuble vers les autres réseaux.
  • Option 6 — les serveurs DNS : à qui le client posera ses questions de noms.
  • Option 15 — le suffixe DNS : ce qu'on ajoute aux noms courts (taper srv-web devient srv-web.formation.local).

Réservations et exclusions

Administrator: Windows PowerShell
# Réservation : SRV-FILE aura TOUJOURS .20 (identifié par sa MAC)
Add-DhcpServerv4Reservation -ScopeId "192.168.100.0" `
                            -IPAddress "192.168.100.20" `
                            -ClientId "00-15-5D-00-00-01" `
                            -Name "SRV-FILE" -Description "Serveur de fichiers"

# Exclusion : ne JAMAIS distribuer .10 à .20 (réservé aux serveurs statiques)
Add-DhcpServerv4ExclusionRange -ScopeId "192.168.100.0" `
                               -StartRange "192.168.100.10" -EndRange "192.168.100.20"

# Bail de 8 jours
Set-DhcpServerv4Scope -ScopeId "192.168.100.0" -LeaseDuration "8.00:00:00"
Réservation contre exclusion : ne pas confondre

Une réservation attribue une IP fixe à une machine via DHCP (la chambre attitrée). Une exclusion retire des adresses de la distribution (chambres réservées au personnel, jamais louées). Une IP statique posée à la main dans la plage DHCP, sans exclusion, provoque tôt ou tard un conflit d'adresses.

Tester côté client

Intégration DNS / DHCP

La réception qui prévient le standard

Quand la réception (DHCP) attribue une chambre à un nouveau client, elle peut prévenir l'annuaire (DNS) : « la machine client-01 est désormais en .50 ». C'est la mise à jour dynamique : les clients DHCP s'inscrivent tout seuls dans le DNS, sans qu'on ait à créer chaque enregistrement à la main.

Administrator: Windows PowerShell
# Autoriser la zone à accepter les mises à jour (mode sécurisé)
Set-DnsServerPrimaryZone -Name "formation.local" -DynamicUpdate Secure

# Dire au DHCP de mettre à jour le DNS à chaque bail
Set-DhcpServerv4DnsSetting -DynamicUpdates Always `
                           -DeleteDnsRROnLeaseExpiry $true `
                           -UpdateDnsRRForOlderClients $true

Get-DhcpServerv4DnsSetting   # vérifier les paramètres

Analysons les paramètres :

  • -DynamicUpdates Always : le DHCP inscrit toujours le client dans le DNS.
  • -DeleteDnsRROnLeaseExpiry $true : quand le bail expire, l'enregistrement DNS est supprimé (l'annuaire reste propre).
  • -UpdateDnsRRForOlderClients $true : prend aussi en charge les anciens clients qui ne savent pas s'inscrire eux-mêmes.
Les valeurs vrai/faux en PowerShell

On écrit $true et $false, avec le dollar et une minuscule. C'est une convention stricte du langage.

Petit script de contrôle global, à faire tourner sur le serveur :

Administrator: Windows PowerShell
$TestResults = @()
$TestResults += "DNS Service : "  + (Get-Service DNS).Status

try   { Resolve-DnsName formation.local -ErrorAction Stop | Out-Null; $TestResults += "DNS Resolution : SUCCESS" }
catch { $TestResults += "DNS Resolution : FAILED" }
$TestResults += "DHCP baux actifs : " + (Get-DhcpServerv4Statistics).InUse
$TestResults | ForEach-Object { Write-Host $_ -ForegroundColor Green }

Diagnostic réseau

Le médecin qui ausculte par étages

Diagnostiquer un réseau, c'est ausculter de l'intérieur vers l'extérieur, comme un médecin qui part du cœur vers les extrémités. On vérifie d'abord que la machine se parle à elle-même, puis qu'elle joint sa passerelle, puis Internet, puis que les noms se résolvent. Dès qu'une étape échoue, on a trouvé l'étage en panne.

Les commandes essentielles

Administrator: Windows PowerShell
ipconfig /all          # configuration IP complète (IP, passerelle, DNS, bail)
ipconfig /flushdns     # vider le cache DNS du client
ping 8.8.8.8           # connectivité brute (sans DNS)
tracert google.com     # le chemin suivi, routeur par routeur
pathping 8.8.8.8       # tracert + statistiques de perte
nslookup formation.local           # interroger le DNS
nslookup -type=MX formation.local  # tester un type précis
netstat -an            # connexions et ports en écoute
arp -a                 # cache des adresses MAC connues

Les équivalents PowerShell

Administrator: Windows PowerShell
Resolve-DnsName formation.local -Type A    # remplace nslookup, plus riche
Test-NetConnection google.com -Port 80     # teste IP + port + route
Get-NetIPConfiguration                     # remplace ipconfig
Get-NetRoute -DestinationPrefix "0.0.0.0/0" # la route par défaut
Get-NetTCPConnection | Where-Object State -eq "Established"
Get-NetNeighbor                            # remplace arp -a
Une commande à NE PAS lancer à l'aveugle

On voit parfois Get-NetAdapter | Disable-NetAdapter pour « réinitialiser » une carte. Sur une session distante, cela coupe votre propre connexion et vous laisse dehors. Ne désactivez une carte qu'en étant physiquement (ou en console) sur la machine.

Outils graphiques

  • Performance Monitor (perfmon) : courbes de débit réseau dans le temps (compteur Network Interface → Bytes Total/sec).
  • Resource Monitor (resmon) : quel processus consomme le réseau, en temps réel (onglet Réseau).

TP guidé 4 — trois pannes à résoudre

Objectif : appliquer la méthode par couches. Durée : 90 min. Pour chaque scénario : identifier l'étage en panne, corriger, valider, documenter.

La méthode, dans l'ordre

Administrator: Windows PowerShell
ping 127.0.0.1          # 1. la machine se parle-t-elle à elle-même ?
ping 192.168.100.1      # 2. joint-elle sa passerelle ?
ping 8.8.8.8            # 3. atteint-elle Internet (par IP) ?
nslookup google.com     # 4. les NOMS se résolvent-ils ?
Lire le résultat

La première étape qui échoue désigne le problème. Internet par IP qui marche mais les noms qui échouent → c'est le DNS. Passerelle injoignable → routage. Connectivité intermittente avec messages d'IP → conflit d'adresses.

Scénario 1 — résolution DNS

Symptôme : les sites sont injoignables par nom, mais l'IP fonctionne. Outils : nslookup, ipconfig /all, Test-DnsServer. Piste : un mauvais serveur DNS dans la configuration du client (option 6 erronée, ou DNS local en panne).

Scénario 2 — conflit d'adresses IP

Symptôme : connectivité intermittente, message « adresse IP en double ». Outils : ping, arp -a, Get-DhcpServerv4Lease. Piste : une IP statique posée à la main dans la plage DHCP, sans exclusion (revoir §8.2).

Scénario 3 — routage

Symptôme : le réseau local fonctionne, mais pas Internet. Outils : tracert, pathping, Get-NetRoute. Piste : passerelle incorrecte ou injoignable (option 3 erronée).

Collecte automatique des informations, à joindre au rapport :

Administrator: Windows PowerShell
ipconfig /all                                   > diagnostic.txt
Get-NetIPConfiguration                         >> diagnostic.txt
Get-DnsClientServerAddress                     >> diagnostic.txt
Get-NetRoute -DestinationPrefix "0.0.0.0/0"    >> diagnostic.txt

Étendue / optionsPlage distribuée + livret d'accueil : option 3 (passerelle), 6 (DNS), 15 (suffixe).
Réservation / exclusionRéservation = IP fixe par MAC ; exclusion = retirer des IP de la distribution.
IntégrationMise à jour dynamique : les clients DHCP s'inscrivent seuls dans le DNS.
DiagnosticPar couches : 127.0.0.1 → passerelle → 8.8.8.8 → nslookup. La 1ʳᵉ étape qui casse désigne la panne.

Réflexes & contrôle

Réflexes anti-erreur
  • Zone inversée → octets du réseau à l'envers + .in-addr.arpa.
  • CNAME → pointe vers un nom, jamais une IP.
  • IP statique dans une plage DHCP → exclusion obligatoire, sinon conflit.
  • $true/$false : dollar + minuscule.
  • Continuation de ligne PowerShell → backtick `, pas l'antislash.
